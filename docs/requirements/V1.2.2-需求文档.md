# FitPlus V1.2.2 需求文档

**版本号**: V1.2.2  
**更新日期**: 2026-05-12  
**状态**: 已上线

---

## 一、版本概述

### 1.1 版本目标
优化用户画像问卷体验，提升动作计划生成质量，完善移动端交互细节。

### 1.2 变更摘要
| 模块 | 变更类型 | 说明 |
|------|----------|------|
| 问卷向导 | 优化 | 滑动条交互、多选逻辑、必填校验 |
| 动作生成 | 优化 | 器械匹配、伤病过滤、模板选择 |
| 移动端体验 | 优化 | 触控响应、手势处理、视觉反馈 |

---

## 二、详细需求

### 2.1 问卷向导（ProfileWizard）

#### 2.1.1 基础信息（Step 1）
| 需求项 | 说明 |
|--------|------|
| 性别选择 | 单选：男/女，默认选中"男" |
| 年龄滑动条 | 范围 18-80，默认 25，步长 1 |
| 身高滑动条 | 范围 140-220cm，默认 170，步长 1 |
| 体重滑动条 | 范围 40-150kg，默认 65，步长 0.1 |
| 目标体重 | 可选，范围同当前体重 |

**移动端滑动条优化需求：**
- 使用原生 touch 事件替代 input range
- 设置 `touchAction: none` 阻止页面滚动干扰
- 触控热区扩大至 48×48pt
- 拖动时移除 CSS transition 实现即时响应
- 按下时滑块放大+阴影增强视觉反馈

#### 2.1.2 健身目标（Step 2）
| 需求项 | 说明 |
|--------|------|
| 健身目标 | **多选**，增肌/减脂/塑形可多选，"维持"独占 |
| 训练经验 | 单选：新手/中级/高级，**无默认** |
| 训练天数 | 滑动条 2-7天，默认 4 |
| 训练时长 | 滑动条 30-120分钟，默认 60 |

**健身目标多选逻辑：**
```
规则1: 增肌(muscle)、减脂(fat_loss)、塑形(shape) 可多选
规则2: 维持(maintain) 独占，选中时其他自动取消
规则3: 选择其他目标时，自动取消"维持"
规则4: 至少选择1个目标才能进入下一步
```

#### 2.1.3 偏好设置（Step 3）
| 需求项 | 说明 |
|--------|------|
| 身体部位偏好 | 多选：胸部/背部/肩部/手臂/腿部/核心 |
| 伤病情况 | 多选：无/肩部/腰部/膝盖/手腕/其他 |
| 可用器械 | 多选：哑铃/杠铃/器械/自重/弹力带/绳索/壶铃，**默认无选中** |
| 训练时间 | 单选：早晨/上午/中午/下午/晚间，**无默认** |

**必填校验规则：**
- 4项配置必须全部有值才能点击"生成我的计划"
- 任一配置为空时，按钮禁用

#### 2.1.4 生成动画
- 3秒进度动画
- 完成后自动跳转到计划页
- 无需确认步骤（Step 4 已移除）

---

### 2.2 动作计划生成

#### 2.2.1 生成逻辑
| 输入条件 | 处理逻辑 |
|----------|----------|
| 用户画像 | 经验 × 器械 = 模板选择 |
| 训练天数 | 2-7天，PPL循环分配 |
| 伤病情况 | 过滤掉涉及伤病部位的动作 |
| 身体偏好 | 优先安排偏好部位动作 |
| 可用器械 | 仅选择可用器械支持的动作 |

#### 2.2.2 模板匹配规则
```
经验等级 × 器械类型 = 动作模板
- 新手 + 哑铃 → beginner_dumbbell
- 新手 + 自重 → beginner_bodyweight
- 中级 + 杠铃 → intermediate_barbell
- ...以此类推
```

#### 2.2.3 输出格式
- 每周计划拆分为每日 TrainingPlan
- 每个计划包含：名称、类型(Push/Pull/Legs)、动作列表
- 每个动作包含：名称、组数、次数、重量、休息时间

## 六、V1.2.2-patch2 迭代需求

### 6.1 首页训练周看板

#### 6.1.1 需求概述
首页展示本周训练计划看板，用户一眼看到今天该练什么，点击即可开始训练。

#### 6.1.2 详细需求
| 需求项 | 说明 |
|--------|------|
| 周看板展示 | 以周为维度展示训练安排（周一~周日），每天显示训练类型（Push/Pull/Legs）或"休息日" |
| 今日高亮 | 当天对应的卡片高亮显示，展示应训练的动作列表摘要 |
| 开始训练 | 今日卡片上有【开始训练】按钮，点击后直接按已生成的计划进入训练页（`/training?planId=xxx`） |
| 数据来源 | 从 `TrainingPlan` 列表中读取，按 `dayOfWeek` 字段匹配星期几 |
| 空状态 | 无计划时显示引导创建入口 |

#### 6.1.3 边界条件
| 场景 | 处理方式 |
|------|----------|
| 同一天有多个计划 | 首页看板展示所有计划，训练类型并列显示 |
| 计划未分配星期几 | 归入"未安排"区域，不展示在周看板中 |
| 修改计划星期几 | 首页看板实时更新 |
| 删除计划 | 首页看板移除对应卡片 |

---

### 6.2 计划 Tab 优化

#### 6.2.1 快速创建支持实时编辑
| 需求项 | 说明 |
|--------|------|
| **目标** | 快速创建过程中就能编辑动作、组数、次数，不需要等创建后再改 |
| **交互流程** | 选类型→选难度→**预览编辑页**（可编辑动作列表）→创建计划 |
| **编辑能力** | 点击任意动作可展开编辑：修改动作（从动作库选择）、修改组数、修改次数 |
| **增删动作** | 支持添加新动作、删除已有动作 |
| **创建** | 点击【创建计划】按用户自定义内容直接保存 |

#### 6.2.2 编辑计划直接进入编辑模式
| 需求项 | 说明 |
|--------|------|
| **目标** | 点击编辑后直接进入可编辑状态，减少操作步骤 |
| **当前问题** | Plans 页点击编辑→跳转 PlanDetail→还要再点编辑按钮 |
| **改进** | 点击编辑后直接以编辑模式打开 PlanDetail 页面（URL 参数控制，如 `/plan/:planId?mode=edit`） |

#### 6.2.3 有计划后支持继续新增
| 需求项 | 说明 |
|--------|------|
| **目标** | 计划页面存在计划后，仍支持再次新增计划 |
| **当前问题** | 有计划后只显示计划列表，没有新增入口 |
| **改进** | 页面顶部或底部增加【+ 添加计划】按钮（类似记录页的添加按钮），点击后弹出创建方式选择（快速创建/智能生成） |

#### 6.2.4 计划支持选择星期几
| 需求项 | 说明 |
|--------|------|
| **目标** | 智能生成和快速创建的计划都支持用户选择训练日 |
| **数据模型** | `TrainingPlan` 新增 `dayOfWeek: number[]` 字段（0=周日, 1=周一, ..., 6=周六） |
| **智能生成** | 生成后默认分配星期几，用户可调整 |
| **快速创建** | 创建过程中增加"选择训练日"步骤 |
| **首页看板** | 根据 `dayOfWeek` 字段匹配展示 |

---

### 6.3 数据模型变更（patch2）

```typescript
// TrainingPlan 新增字段
interface TrainingPlan {
  // ...现有字段
  dayOfWeek: number[];  // 训练日，0=周日 1=周一 ... 6=周六，空数组表示未安排
}
```
